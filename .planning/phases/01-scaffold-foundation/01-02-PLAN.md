# Plan 01-02: CSS Variable Design System

## Goal
Implement the complete CLEANMess design system using CSS variables. This is the single source of truth for all colors, spacing, typography, shadows, and radii used everywhere in the app.

## Context
All decisions are locked in `.planning/phases/01-scaffold-foundation/01-UI-SPEC.md`. Do NOT deviate from the values defined there. This file must be read before writing any CSS.

<task id="01-02-C">
<title>Update app/layout.tsx with Inter font and CSS imports</title>
<action>
Update `src/app/layout.tsx` to:
1. Load Inter font via `next/font/google`
2. Import both CSS files (tokens first, then globals)
3. Apply the font CSS variable to the `<html>` element
4. Set proper metadata for SEO

```tsx
import type { Metadata } from "next";
import { Inter } from "next/font/google";
import "@/styles/tokens.css";
import "@/styles/globals.css";

const inter = Inter({
  subsets: ["latin"],
  variable: "--font-inter",
  display: "swap",
});

export const metadata: Metadata = {
  title: "CLEANMess — Know What's In Your Mess",
  description:
    "AI-powered food and water quality monitoring for college canteen safety. Real-time sensor alerts, historical trends, instant incident reporting.",
  keywords: ["food safety", "IoT", "canteen monitoring", "college mess", "water quality"],
  openGraph: {
    title: "CLEANMess — Know What's In Your Mess",
    description:
      "AI-powered food and water quality monitoring for college canteen safety.",
    type: "website",
  },
};

export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html lang="en" className={inter.variable}>
      <body>{children}</body>
    </html>
  );
}
```

Remove the existing `import "./globals.css"` if present from the auto-generated layout — it is replaced by the two explicit imports above.
