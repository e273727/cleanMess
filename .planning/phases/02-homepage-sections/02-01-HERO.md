# Phase 2.1: Hero Section

## Objective
Create a stunning, high-impact hero section that immediately communicates the CLEANMess value proposition. Must include compelling tagline, clear CTA, and an engaging animated visual (waveform or sensor visualization).

## Requirements Addressed
- HERO-01: Hero section exists and is visible on page load
- HERO-02: Tagline communicates product value
- HERO-03: CTA button present and functional (links to #problem or contact)
- HERO-04: Visual element (animated waveform/sensor) enhances the message

## Component Structure
```
src/app/page.tsx
  └─ HeroSection component
     ├─ Heading (h1 tagline)
     ├─ Subheading (supporting copy)
     ├─ CTA Button (primary action)
     └─ AnimatedVisual (waveform or sensor animation)
```

## Content Guidelines
- **Tagline**: Short, punchy, value-focused (e.g., "Real-Time Intelligence for Food Safety")
- **Subheading**: 2–3 sentences explaining the core problem it solves
- **CTA Copy**: "Explore How It Works" or "See the Dashboard"
- **Visual**: Animated SVG waveform (pH/turbidity/temp lines) OR sensor icon with pulsing animation

## Technical Details
- Use CSS Modules for styling (HeroSection.module.css)
- Hero section fills 100dvh (full viewport height)
- Use CSS scroll-snap-align: start
- Optional: Simple Framer Motion for waveform animation (if not using CSS animations)
- Responsive: On mobile, reduce text size; visual should remain prominent

## Success Criteria
1. Hero section displays full viewport height with no overflow
2. Text and visual are centered and readable on mobile (375px) and desktop (1440px)
3. CTA button is clickable and smooth-scrolls to target section
4. Animated visual (waveform or sensor) is smooth and loops without janky performance
5. No console errors on load
